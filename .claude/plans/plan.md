# План улучшения отрисовки формирующихся свечей

## Цель
Добавить визуальную индикацию формирующихся свечей и улучшить UX до уровня профессиональных платформ (TradingView, Scalpboard.io).

## Анализ текущей реализации

### Архитектура
- **Библиотека**: lightweight-charts v5.2.0
- **Источники данных**: 
  - `useWsCandle` - получает готовые свечи с сервера (с флагом `isFinal`)
  - `useWsTrade` - строит свечи из трейдов в реальном времени
- **Рендеринг**: RAF-батчинг с throttling 16ms (60fps)
- **Сервер**: уже отправляет флаг `isFinal: !!k.x` (binance-futures.ts:342, binance-spot.ts:335)

### Ограничения lightweight-charts
Lightweight-charts v5.2.0 **не поддерживает** прямую стилизацию отдельных свечей через API:
- Нет `customValues` для candlestick series
- Нет способа задать opacity для конкретной свечи
- Все свечи используют одинаковые `upColor`/`downColor`

### Возможные подходы

#### ❌ Подход 1: Использовать customValues (НЕ РАБОТАЕТ)
```typescript
// Это НЕ работает в lightweight-charts v5.2.0
candleRef.current.update({
  time: c.time,
  open: c.open,
  high: c.high,
  low: c.low,
  close: c.close,
  customValues: { isForming: true } // ❌ Игнорируется
})
```

#### ❌ Подход 2: Custom Series View (СЛОЖНО)
- Требует создания полностью кастомного рендерера
- Нужно переписать всю логику отрисовки свечей
- Риск багов и проблем с производительностью
- Оценка: 20+ часов работы

#### ✅ Подход 3: CSS-анимация + индикаторы (ОПТИМАЛЬНО)
Использовать визуальные индикаторы вокруг графика:
1. **Live-индикатор** - показывает, что данные обновляются
2. **Stale data warning** - предупреждает об отсутствии обновлений
3. **Flash-эффект** - подсвечивает область графика при значительных изменениях
4. **Адаптивный throttling** - ускоряет обновления при высокой волатильности

## Выбранный подход: Комбинированное решение

### Фаза 1: Визуальные индикаторы (Приоритет 1)
**Время: 2-3 часа**

1. **Live-индикатор** в углу графика
   - Зеленая точка + "LIVE" когда данные обновляются
   - Серая точка + "PAUSED" когда нет обновлений >5 секунд
   - Пульсирующая анимация

2. **Stale data overlay**
   - Полупрозрачный overlay "Переподключение..." через 5 секунд без данных
   - Автоматически исчезает при возобновлении

3. **Flash-эффект при значительных изменениях**
   - Короткая вспышка границы графика при изменении >0.5%
   - Зеленая для роста, красная для падения

### Фаза 2: Адаптивный throttling (Приоритет 2)
**Время: 1-2 часа**

1. **Расчет волатильности**
   - Скользящее окно 20 свечей
   - Средний % изменения между свечами

2. **Динамический интервал**
   - Высокая волатильность (>2%): 8ms
   - Средняя волатильность (1-2%): 12ms
   - Низкая волатильность (<1%): 16ms

3. **Приоритетные обновления**
   - Новые high/low рендерятся немедленно, игнорируя throttling

### Фаза 3: Улучшенная синхронизация (Приоритет 3)
**Время: 1 час**

1. **Приоритет kline над trade**
   - Trade-построенные свечи помечаются флагом `source: 'trade'`
   - При получении kline с тем же timestamp - немедленная замена
   - Логирование расхождений для отладки

### Новые hooks

#### useLiveIndicator
```typescript
function useLiveIndicator(
  candlesDataRef: React.RefObject<UnifiedCandle[]>
): { isLive: boolean; lastUpdate: number }
```
- Отслеживает timestamp последнего обновления
- Возвращает isLive=false если >3 секунд без обновлений

#### useStaleDataDetection
```typescript
function useStaleDataDetection(
  lastUpdateRef: React.RefObject<number>,
  threshold?: number
): boolean
```
- Проверяет каждую секунду
- Возвращает true если прошло >threshold мс без обновлений

#### useFlashEffect
```typescript
function useFlashEffect(
  candlesDataRef: React.RefObject<UnifiedCandle[]>,
  threshold?: number
): 'up' | 'down' | null
```
- Отслеживает изменения последней свечи
- Триггерит flash при изменении >threshold%
- Автоматически сбрасывается через 300ms

#### useAdaptiveThrottling
```typescript
function useAdaptiveThrottling(
  candlesDataRef: React.RefObject<UnifiedCandle[]>
): number
```
- Рассчитывает волатильность
- Возвращает оптимальный интервал throttling

## Риски и митигация

### Риск 1: Производительность
**Проблема**: Дополнительные проверки и hooks могут замедлить рендеринг

**Митигация**:
- Все проверки в useRef, не вызывают re-render
- Throttling для проверок (1 раз в секунду для stale detection)
- RAF для flash-эффектов

### Риск 2: Визуальный шум
**Проблема**: Слишком много индикаторов могут отвлекать

**Митигация**:
- Минималистичный дизайн
- Полупрозрачность
- Индикаторы появляются только когда нужны

### Риск 3: Ложные срабатывания stale detection
**Проблема**: На низколиквидных парах могут быть долгие паузы между трейдами

**Митигация**:
- Порог 5 секунд (достаточно большой)
- Отдельная проверка WebSocket.readyState
- Показывать только если WS подключен, но данных нет

3. **Высокая волатильность**
   - Throttling уменьшается до 8ms
   - Обновления более частые и плавные
   - Flash-эффекты чаще

4. **Низколиквидная пара**
   - Throttling увеличивается до 16ms
   - Нет ложных stale warnings
   - Live-индикатор корректно работает

## Альтернативы (отклонены)

### Альтернатива 1: Обновить lightweight-charts до v6
**Причина отклонения**: 
- v6 еще в beta
- Может сломать существующий код
- Не гарантирует поддержку нужных фич

### Альтернатива 2: Переход на другую библиотеку (TradingView, Chart.js)
**Причина отклонения**:
- Слишком большой объем работы
- Потеря существующей функциональности
- TradingView платный

### Альтернатива 3: Canvas overlay для кастомной отрисовки
**Причина отклонения**:
- Сложная синхронизация с lightweight-charts
- Проблемы с масштабированием и скроллингом
- Дублирование логики рендеринга

## Итоговая оценка

**Общее время**: 4-6 часов

**Приоритеты**:
1. Фаза 1 (визуальные индикаторы) - 2-3 часа - **КРИТИЧНО**
2. Фаза 2 (адаптивный throttling) - 1-2 часа - **ВАЖНО**
3. Фаза 3 (улучшенная синхронизация) - 1 час - **ЖЕЛАТЕЛЬНО**

**Результат**:
- Пользователь видит, что данные обновляются в реальном времени
- Предупреждение при проблемах с подключением
- Визуальный feedback при значительных изменениях цены
- Более плавные обновления при высокой волатильности
- UX на уровне профессиональных платформ
