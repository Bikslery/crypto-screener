# План исправления пустых графиков на таймфреймах 4Ч и 1Ч

## 🔍 Анализ проблемы

### Корневая причина
При выборе таймфреймов **4Ч** и **1Ч** графики становятся пустыми из-за следующих проблем:

1. **Таймфрейм 4Ч (4h)**:
   - ❌ НЕ включен в предзагрузку (`PRELOAD_TFS` в `preload.ts:5`)
   - ❌ НЕ подписан на WebSocket обновления
   - ❌ НЕ включен в периодическое обновление
   - ✅ Поддерживается биржевыми адаптерами
   - ✅ Определен в типах клиента и сервера

2. **Таймфрейм 1Ч (1h)**:
   - ✅ Включен в предзагрузку (`PRELOAD_TFS`)
   - ❌ НЕ подписан на WebSocket обновления (`wsTfs = ['5m', '1m']`)
   - ❌ НЕ включен в периодическое обновление (`refreshTfs = ['5m', '1m', '15m']`)
   - ✅ Поддерживается биржевыми адаптерами

### Проблемные файлы
- `server/src/services/candles/preload.ts` - основная проблема
- `client/src/components/charts/ChartGrid.tsx` - может требовать улучшения обработки ошибок

## 🛠️ План исправления

### Этап 1: Критические исправления (ПРИОРИТЕТ 1)

#### 1.1 Исправить preload.ts
**Файл:** `server/src/services/candles/preload.ts`

**Изменения:**
```typescript
// Строка 5: Добавить 4h в предзагрузку
export const PRELOAD_TFS = ['5m', '15m', '1h', '4h'] as const

// Строка 91: Добавить 1h и 4h в WS подписки
const wsTfs = ['5m', '1m', '1h', '4h']

// Строка 108: Добавить 1h и 4h в периодическое обновление
const refreshTfs = ['5m', '1m', '15m', '1h', '4h']
```

**Обоснование:**
- 4h нужен для долгосрочного анализа
- 1h - популярный таймфрейм для свинг-трейдинга
- Без WS подписок графики не обновляются в реальном времени
- Без периодического обновления данные устаревают

**Риски:**
- Увеличение нагрузки на сервер (больше WS подписок)
- Увеличение использования памяти (больше кэшированных свечей)

**Митигация:**
- Уменьшить количество предзагружаемых символов для 4h
- Оптимизировать кэш (см. Этап 2)

#### 1.2 Добавить обработку ошибок в ChartGrid.tsx
**Файл:** `client/src/components/charts/ChartGrid.tsx`

**Проблема:** При пустом ответе от API нет информативного сообщения пользователю

**Изменения:**
- Добавить состояние ошибки в `useFullHistory`
- Показывать сообщение "Нет данных для этого таймфрейма" вместо пустого графика
- Логировать ошибки для отладки

### Этап 2: Оптимизация производительности (ПРИОРИТЕТ 2)

#### 2.1 Умная предзагрузка
**Файл:** `server/src/services/candles/preload.ts`

**Текущая проблема:**
- Все таймфреймы загружаются для всех топ-100 символов
- 4h используется реже, чем 5m/15m
- Избыточная нагрузка на API бирж

**Решение:**
```typescript
// Разные лимиты для разных таймфреймов
const PRELOAD_LIMITS = {
  '5m': 100,   // топ-100 символов
  '15m': 100,  // топ-100 символов
  '1h': 50,    // топ-50 символов
  '4h': 30,    // топ-30 символов
}
```

**Преимущества:**
- Снижение нагрузки на API
- Быстрее стартует сервер
- Меньше памяти на кэш

#### 2.2 Оптимизация кэша свечей
**Файл:** `server/src/services/candles/candle-cache.ts`

**Текущие лимиты:**
```typescript
const MAX_CANDLES_PER_KEY = 2000
const MAX_TOTAL_CANDLES = 1_500_000
```

**Проблема:**
- Для 4h нужно меньше свечей (2000 свечей = 333 дня)
- Для 1m нужно больше свечей для backfill

**Решение:**
```typescript
const MAX_CANDLES_BY_TF: Record<string, number> = {
  '1m': 2000,   // ~33 часа
  '5m': 2000,   // ~7 дней
  '15m': 2000,  // ~20 дней
  '1h': 1000,   // ~41 день
  '4h': 500,    // ~83 дня
  '1d': 365,    // ~1 год
  '1w': 104,    // ~2 года
}
```

#### 2.3 Ленивая загрузка для редких таймфреймов
**Файл:** `client/src/components/charts/ChartGrid.tsx`

**Идея:**
- 1m, 5m, 15m - предзагружаются всегда
- 1h, 4h - загружаются по требованию (on-demand)
- 1d, 1w - загружаются по требованию

**Реализация:**
- Добавить флаг `enableBackfill` в зависимости от таймфрейма
- Для 4h/1d/1w использовать меньший `limit` в начальной загрузке

## ⚠️ Потенциальные проблемы и решения

### Проблема 1: Увеличение нагрузки на API
**Симптом:** 429 ошибки от Binance после добавления 4h в предзагрузку

**Решение:**
- Увеличить `RATE_LIMIT_MS` в preload.ts с 50ms до 100ms
- Уменьшить количество символов для 4h (см. умную предзагрузку)
- Использовать rate limiter более агрессивно

### Проблема 2: Увеличение использования памяти
**Симптом:** Сервер использует > 2GB RAM

**Решение:**
- Уменьшить `MAX_TOTAL_CANDLES` с 1.5M до 1M
- Реализовать оптимизацию кэша по таймфреймам
- Чаще вызывать `evictIfNeeded()`

### Проблема 3: Медленная загрузка 4h для старых монет
**Симптом:** Backfill для 4h занимает > 10 секунд

**Решение:**
- Уменьшить `MAX_BACKFILL_CANDLES` для 4h
- Увеличить `BACKFILL_CONCURRENCY` для 4h
- Показывать прогресс пользователю

### Проблема 4: Пустые данные для новых листингов
**Симптом:** Новые монеты не имеют истории на 4h

**Решение:**
- Показывать сообщение "Недостаточно истории для этого таймфрейма"
- Предлагать переключиться на 1h или 5m
- Автоматически переключаться на доступный таймфрейм
