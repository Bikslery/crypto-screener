# Исправление ленивой догрузки свечей

## Проблема
При скролле графика к левому краю (в прошлое) пользователь видел пустое место — свечи не догружались автоматически.

## Корневая причина
В `useLazyScroll` (ChartGrid.tsx) был слишком маленький порог для триггера загрузки:
- **Старый порог**: фиксированные 50 баров
- **Проблема**: при быстром скролле пользователь доходил до края (`range.from = 0`) до того, как данные успевали загрузиться

## Исправления

### 1. Динамический порог загрузки
```typescript
// Было:
if (range.from > 50) return

// Стало:
const visibleBars = range.to - range.from
const threshold = Math.max(100, visibleBars * 0.3)
if (range.from > threshold) return
```

**Логика**: загрузка начинается когда пользователь находится в пределах 30% от видимого диапазона или минимум 100 баров (что больше). Это дает достаточно времени для загрузки данных до того, как пользователь увидит пустое место.

### 2. Визуальный индикатор загрузки
Добавлен индикатор в левом верхнем углу графика, который показывается во время догрузки исторических данных:
```typescript
{!isInitialLoading && isLoadingMore && (
  <div className="absolute top-[8px] left-[8px] z-30">
    <div className="flex items-center gap-2 px-3 py-1.5 rounded-[4px] bg-[#1a1a1a]/95">
      <div className="w-[12px] h-[12px] border-2 border-[#555] border-t-[#ccc] rounded-full animate-spin" />
      <span className="text-[11px] text-[#aaa]">Загрузка истории...</span>
    </div>
  </div>
)}
```

### 3. Улучшенная обработка состояния загрузки
- Добавлен параметр `setIsLoadingMore` в `useLazyScroll`
- Состояние загрузки корректно устанавливается/сбрасывается во всех точках выхода
- Пользователь видит когда идет загрузка

## Как работает сейчас

1. **Пользователь скроллит влево** (в прошлое)
2. **Когда `range.from` становится ≤ порога** (30% от видимого диапазона или 100 баров):
   - Показывается индикатор "Загрузка истории..."
   - Запрашивается 1000 старых свечей через `getOrFetchOlder`
3. **Данные загружаются с сервера** (используется chunk-based система с Redis кешем)
4. **Новые свечи добавляются в начало** массива
5. **Видимый диапазон корректируется** чтобы пользователь остался на том же месте
6. **Индикатор скрывается**

## Поведение как на scalpboard.io

✅ Свечи догружаются **до** того как пользователь видит пустое место  
✅ Плавная загрузка без скачков  
✅ Визуальная обратная связь (индикатор загрузки)  
✅ Динамический порог адаптируется к масштабу графика

## Тестирование
1. Откройте любой график в полноэкранном режиме
2. Скроллите влево (в прошлое)
3. Когда приблизитесь к краю — увидите индикатор "Загрузка истории..."
4. Свечи догрузятся автоматически, пустого места не будет
