### Как протестировать:

#### 1. Live-индикатор
1. Откройте http://localhost:5173
2. Наблюдайте зеленую пульсирующую точку "LIVE" в правом верхнем углу графиков
3. Отключите интернет → через 3 секунды индикатор станет серым "PAUSED"
4. Включите интернет → индикатор снова станет зеленым "LIVE"

#### 2. Stale Data Warning
1. Отключите интернет
2. Через 5 секунд появится overlay "Переподключение к серверу..."
3. Включите интернет → overlay исчезнет

#### 3. Flash-эффект
1. Наблюдайте за графиками во время активной торговли
2. При резких движениях цены (>0.5%) увидите вспышку границы графика
3. Зеленая вспышка = рост, красная = падение

#### 4. Адаптивный throttling
1. Откройте DevTools → Console
2. Наблюдайте за плавностью обновлений
3. При высокой волатильности обновления будут более частыми (8ms)
4. При низкой волатильности - реже (16ms)

#### 5. Логирование расхождений
1. Откройте DevTools → Console
2. Наблюдайте за сообщениями типа:
   ```
   [useWsCandle] Kline replaced trade-built candle {
     symbol: "BTCUSDT",
     tf: "1m",
     time: 1234567890,
     tradClose: 50000.5,
     klineClose: 50001.2,
     diffPct: "0.14"
   }
   ```

## 🔧 Техническая реализация

### Архитектурные решения:

1. **Hooks-based подход**
   - Все новые функции реализованы как React hooks
   - Легко тестируются и переиспользуются
   - Не нарушают существующую архитектуру

2. **Ref-based state**
   - Использование `useRef` вместо `useState` где возможно
   - Избегание лишних re-renders
   - Оптимальная производительность

3. **CSS-анимации**
   - Аппаратное ускорение через GPU
   - Плавные 60fps анимации
   - Минимальная нагрузка на CPU

4. **Адаптивность**
   - Динамическая подстройка под условия рынка
   - Приоритизация важных обновлений
   - Баланс между плавностью и производительностью
